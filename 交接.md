## 專案概觀

**這是什麼**：一個 Next.js 14 portfolio site，用來展示作者另一個 repo [`_quant-terminal`](https://github.com/GKS711/_quant-terminal)（Python 量化系統）的 output。不是「自己跑量化」的 web app，是「把 Python 後端結果視覺化的個人作品集」。

**端到端能做什麼**：
- 12 檔股票（4 國市場：TW/US/HK/CN）
- 每檔有真實 Yahoo Finance 報價（30 天 OHLC）
- 11 種策略訊號（純函式 / 132 cells / 真實計算）
- Gemma 4 31B 為每檔生 AI 決策 JSON（buy/hold/sell + 0–100 信心 + rationale + risks + catalysts）
- AI 顧問 chat（advisor section）能即時 token-by-token 串流，回答受 12 檔 snapshot 的真實數據約束

## 規則與限制（用過會後悔）

### 🚨 Yahoo Finance 反爬規則
- **Node fetch 會被 Yahoo 直接 ban**（429 + IP 黑名單）
- 已知唯一可用的方式：`scripts/refresh-data.mjs` 裡的 `execSync('curl ...')` shell-out，搭配：
  - 短 UA：`Mozilla/5.0 (Macintosh) AppleWebKit/537.36`
  - host 用 `query2.finance.yahoo.com`（不是 query1）
  - 12 檔**序列**抓（不能並行），每檔間 4–6 秒 delay
- `yahoo-finance2 v2.14.0` 已棄用 historical API，**不要試圖回去用 npm 套件**

### 🚨 Gemma 4 對話格式
- **history 必須以 user role 開頭**——前端如果先送 model role 訊息，Gemma 會 throw
- `app/api/chat/route.ts` 已有 `cleanedHistory.filter(...)` 把 leading 非 user role 的訊息丟掉
- thinking model 第一個 token 慢（10-30s）, **`maxOutputTokens` 至少 4096**（之前設 700 被截斷一半）

### 🚨 PPT 截圖比例不能壓縮
- Codex 評分對「圖片被壓扁」非常敏感
- 規則：圖盒 aspect ratio 必須 ≈ 圖片本身比例（誤差 <5%）
- 例：`10b-nvda-hero.png` 是 2880×1200（2.4:1），盒子要設 9×3.75 inch（2.4:1），不能設 9×1.7（5.29:1 → 壓扁）
- pptxgenjs 用 `sizing: { type: "contain", w, h }` 才會等比

### 🚨 Puppeteer 抓單一元素
- 不要用 `page.screenshot({ clip: ... })`——`clip` 是 viewport 座標，scroll 後會錯位
- 正確做法：`page.evaluateHandle(() => document.querySelector(...).closest('.target-class'))` → `elem.asElement().screenshot()`，puppeteer 會自動處理 scroll
- 範例見 `scripts/screenshot-extras.mjs` 的 chat panel 截法

### 設計準則（已被 user 接受）
- 配色：黑底（#0A0A0A）+ 一個強調色「mint #4EAA85」
- 數字一律 `tabular-nums`
- 標題 weight 510（非 500 非 600）
- 動畫不超過 0.4Hz 慢脈衝
- 詳見 `DESIGN.md`
